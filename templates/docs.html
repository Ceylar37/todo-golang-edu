Тут будет краткое описание того как работать с этим API

  • /api/ [GET]

    Этот ендпоинт возвращает все todo, которые лежат в бд Возвращается json со следующими полями
    Пример:
    {
      "result": [
          {
            "id": 1,
            "description": "some description",
            "isDone": false,
            "isFavourite": true
          }
      ]
    }

  • /api/ [POST]

    Этот ендпоинт принимает объект с полем: description
    Пример:
    {
      "description": "some description for new todo",
    }
    И создаёт новую todo на основе description
    Возвращается json в которой будет id созданной todo
    Пример:
    {
      "id": 12
    }
     При этомотправлять поля isDone и isFavourite не обязательно, тогда они будут созданы со стандартными значениями false

  • api/changeIsDone/:id [PUT]

    Это динамический эндпоинт в его URL параметр :id значит id изменяемой todo
    Этот ендпоинт принимает объект с полем isDone и меняет значение isDone у todo с переданным в URL id
    Пример:
    {
      "isDone": true
    }
    Возвращается json в которой будет id изменённой todo
    Пример:
    {
      "id": 12
    }

  • api/changeIsFavourite/:id [PUT]

    Этот эндпоинт работает так же как и предыдущий, но изменяет поле isFavourite
    Пример:
    {
      "isFavourite": true
    }
    Возвращается json в которой будет id изменённой todo
    Пример:
    {
      "id": 12
    }

  • api/:id [DELETE]

    Этот эндпоинт так же динамический, но он удаляет todo с переданным в URL id
    Возвращается json в которой будет id удалённой todo
    Пример:
    {
      "id": 12
    }
